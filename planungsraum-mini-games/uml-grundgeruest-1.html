Grundstruktur eines Klassendiagramms

In der objektorientierten Programmierung ist eine Klasse wie ein Bauplan. Im sogenannten Klassendiagramm zeichnen wir diesen Bauplan auf – damit wir genau sehen können, welche Eigenschaften (Attribute) und welche Fähigkeiten (Methoden) ein Objekt haben soll.

So ist ein Klassendiagramm aufgebaut:

Oben steht der Klassenname, z.B. Roboter
In der Mitte stehen die Attribute, z.B. farbe und matrialart
Unten stehen die Methoden, z.B. fliegen() und bewegeDich()

So sieht das Grundgerüst einer Beispiel-Klasse im Diagramm aus:

Roboter
- farbe: String
- materialart: String
+ fliegen(): boolean
+ bewegeDich(): boolean

Beziehungen zwischen Klassen

Klassen können miteinander in Beziehung stehen – zum Beispiel, wenn ein Objekt das andere kennt oder nutzt. Solche Verbindungen nennt man Assoziationen.

Eine binäre Assoziation verbindet genau zwei Klassen. Im UML-Diagramm wird sie durch eine einfache Linie dargestellt.

Beispiel: Eine Pflanze und ein Roboter sind verbunden, weil Die Pflanze den Roboter steuert.

Pflanze
- wasserbedarf: String
+ wachse(): void
Roboter
- farbe: String
+ fliegen(): boolean

Diese Linie zeigt eine einfache Beziehung zwischen den Klassen. Man sagt auch: Sie stehen in einer Assoziation.

Übung: Erkenne die Assoziation

Klicke auf zwei Klassen, die miteinander in Beziehung stehen sollen.

Dabei sollen folegende Beziehungen dargestellt werden:
