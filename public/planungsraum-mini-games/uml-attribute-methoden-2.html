Modifizierer entdecken

Klicke auf eine Karte, um zu sehen, wie und wo der Modifizierer eingesetzt wird:

public
Beschreibung: Überall sichtbar
Anwendbar auf: Klassen, Methoden, Attribute
Beispiel:
+ setName(name: String): void
private
Beschreibung: Nur innerhalb der Klasse
Anwendbar auf: Methoden, Attribute
Beispiel:
- alter: int
protected
Beschreibung: In Klasse & Subklassen sichtbar
Anwendbar auf: Methoden, Attribute
Beispiel:
# groesse: double
static
Beschreibung: Gehört zur Klasse
Anwendbar auf: Methoden, Attribute
Beispiel:
- static anzahl: int
final
Beschreibung: Nicht veränderbar/überschreibbar
Anwendbar auf: Attribute, Methoden, Klassen
Beispiele:
final class Konstante { }
final int MAX = 10;
abstract
Beschreibung: Nur Signatur, keine Umsetzung
Anwendbar auf: Klassen, Methoden
Beispiele:
abstract class Pflanze { }
abstract void bewegeDich();

Übung: Modifizierer verstehen

Fülle die Lücken mit den passenden Modifizierern aus.

Der Modifizierer erlaubt den Zugriff von überall.

Der Modifizierer erlaubt den Zugriff nur innerhalb der eigenen Klasse.

Mit können auch Subklassen auf das Element zugreifen.

Wenn eine Methode nicht verändert werden darf, wird sie mit markiert.

Eine Methode, bei der nur der Methodenkopf bekannt ist, aber keine Implementierung, ist .

Der Modifizierer bedeutet, dass das Element zur Klasse gehört und nicht zu einzelnen Objekten.
