Willkommen zur Lerneinheit: Methoden und Attribute

In dieser kurzen Einheit lernst du den Unterschied zwischen Attributen und Methoden kennen.

Außerdem wirst du spielerisch üben, wie man sie den richtigen Klassen zuordnet.

Beginnen wir mit den Basics!

Grundbegriffe:
In der objektorientierten Programmierung beschreiben wir mit Klassen allgemeine Baupläne für Dinge – z.B. für Roboter oder Pflanzen.
Wenn wir aus einer Klasse ein konkretes Exemplar erzeugen - z.B Wasser-Roboter, spricht man von einem Objekt oder einer Instanz.

Attribut – die Eigenschaft eines Objekts
Ein Attribut beschreibt den Zustand oder eine Eigenschaft eines Objekts.
Beispiel: Die Klasse Roboter besitzt das Attribut farbe.
Ein konkreter Roboter könnte dann z.B. die Farbe grün haben.
→ farbe ist ein Attribut.

Methode – das Verhalten eines Objekts
Eine Methode beschreibt ein Verhalten oder eine Funktion, die ein Objekt ausführen kann.
Beispiel: Die Klasse Roboter enthält die Methode spreche().
Ein konkreter Roboter kann dann z. B. roboter.spreche() aufrufen.
→ spreche() ist eine Methode.

Kurzfassung:
– Attribute = Eigenschaften (z. B. farbe)
– Methoden = Fähigkeiten (z. B. spreche())

Zugriffsmodifizierer: Public und Private

In objektorientierten Programmiersprachen steuern wir mit sogenannten Zugriffsmodifizierern, wer auf bestimmte Bestandteile einer Klasse zugreifen darf.

Das machen wir auch in unserem Klassendiagramm kenntlich.

Die beiden wichtigsten Modifizierer sind:

public

  • Das Element ist von außen sichtbar
  • Instanzen anderer Klassen können direkt darauf zugreifen
  • Im Klassendiagramm wird es durch ein Pluszeichen + dargestellt
  • Beispiel: + wachse()
  • Oft sind Methoden public, da sie das Verhalten eines Objekts nach außen bereitstellen und anderen Klassen ermöglichen, gezielt darauf zuzugreifen

private

  • Das Element ist nur innerhalb der eigenen Klasse sichtbar
  • Von außen ist kein direkter Zugriff möglich
  • Im Klassendiagramm wird es durch ein Minuszeichen – dargestellt
  • Beispiel: - farbe
  • Oft sind Attribute private, um den internen Zustand vor ungewolltem Zugriff zu schützen – das nennt man Kapselung

Damit man trotzdem kontrolliert auf private Attribute zugreifen kann, nutzt man sogenannten Methoden Getter und Setter:

  • Ein Getter gibt den aktuellen Wert eines privaten Attributs (z.B. - farbe) zurück
    Beispiel: + getFarbe()
  • Ein Setter ändert den Wert eines privaten Attributs kontrolliert
    Beispiel: + setFarbe(neueFarbe)
  • Getter und Setter sind public, damit andere Klassen gezielt und sicher auf Attribute zugreifen können

Datentypen in Klassen

In Klassen verwenden wir verschiedene Datentypen, um den Wertbereich von Attributen und Rückgabewerten zu beschreiben.

Hier sind einige der wichtigsten grundlegenden Datentypen:

  • int – ganze Zahl (z.B. 15, -2)
  • float oder double – Kommazahl (z.B. 3.14, -0.5)
  • boolean – wahr oder falsch (true / false)
  • String – Text (z.B. "grün" oder "Roboter-X")

Beispiele für die Schreibweise der Attribute und Methoden mit Typ im Klassendiagramm:

  • + getfarbe(): String
  • - anzahlBlaetter: int
  • - roboterGroesse: double

Methoden genauer erklärt

Eine Methode beschreibt eine Fähigkeit oder ein Verhalten eines Objekts – wie fliegen() oder spreche().

Methoden haben oft:

  • Einen Rückgabetyp – das ist der Typ des Werts, den die Methode zurückliefert (z. B. String, int, void für "nichts zurückgeben")
  • Parameter – Werte, die beim Aufruf übergeben werden (in den Klammern)

Beispiel:

public String spreche(String wort)
  • public → sichtbar von außen
  • String → Rückgabetyp
  • spreche → Methodenname
  • (String wort) → die Methode bekommt ein Wort übergeben

Konstruktoren

Ein Konstruktor ist eine besondere Methode, die automatisch aufgerufen wird, wenn ein neues Objekt erzeugt wird. Er hat:

  • Keinen Rückgabetyp
  • Immer denselben Namen wie die Klasse

Beispiel:

public Roboter(String farbe, float groesse)
⟶ Diese Methode erzeugt einen neuen Roboter mit Farbe und Größe.
