Interfaces in Klassendiagrammen

Ein Interface ist eine Art Vertrag: Es beschreibt, welche Methoden eine Klasse bereitstellen muss, ohne die konkrete Implementierung anzugeben.

Im Klassendiagramm erkennst du ein Interface an dem Schlüsselwort «interface» über dem Klassen-Namen.

Eine Klasse wie Kampf-Roboter kann das Interface Roboter verwenden, um zu sagen: „Ich verspreche, dass ich alle diese Fähigkeiten besitze – ich kann spucken, mich bewegen und kämpfen.“

Die gestrichelte Linie mit offener Pfeilspitze zeigt immer zur Interface-Klasse. So erkennt man: Diese Klasse erfüllt die Anforderungen des Interfaces.

Ein Interface selbst hat keine ausgehenden Pfeile, weil es nur vorgibt, was andere Klassen können müssen – es kennt die Klassen aber nicht.

+ spucken(): void
+ bewegen(): void
+ kämpfen(): void

Interface-Inspector – FlipCard

Nur eine Klasse implementiert das Interface MedBot korrekt. Klicke auf eine Karte, um sie umzudrehen und deinen Verdacht zu überprüfen!

«interface» MedBot
+ messen(): float
+ desinfizieren(): void
MedBot A
- reinigungsStatus: int
+ messen(): float
+ desinfizieren(): void
✅ Diese Klasse implementiert das Interface korrekt.
MedBot B
- reinigungsStatus: int
+ messen(): int
+ desinfizieren(): void
❌ Fehler: messen() hat den falschen Rückgabetyp (int statt float).
MedBot C
- messen: float
- reinigungsStatus: int
+ desinfizieren(): void
❌ Fehler: messen ist ein Attribut, aber das Interface verlangt eine Methode.
💬 Tipp: Achte immer genau auf Klammern – sie machen den Unterschied zwischen Methode und Attribut!
